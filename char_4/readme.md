# 第四章 同步并发操作
## 精读:
    全书最核心,最实用的一章,解决线程之间如何等待,通知,传递结果.
### 4.1 条件变量condition_variable
- 作用:线程a等待,线程b通知
- 两个核心:
    - wait(lock,条件):没满足就休眠
    - notify_one() / notify_all():唤醒等待线程
- 典型用途: 生产者-消费者模型

### 4.2 future一次性事件
- std::future: 获取异布任务结果
- std::promise: 手动设置值 / 异常,作用手动控制线程就诶过,在线程中提交结果
- std::packaged_task: 包装任务,绑定future, 原理:调用任务->执行函数->结果自动写入future. 作用:用于异步任务封装,线程持,任务队列核心组件
- std::async: 直接启动异步任务,返回future,原理:库自动创建线程/调度->运行函数->结果存入future. 作用:最简单的异步执行方式

### 4.3 超时等待
- wait_for: 相对时间
- wait_until:绝对时间

### 4.4 高频使用模式
- 生产者-消费者
- 异步计算
- 线程间传值 / 传异常

# 第三章 线程间共享数据
## 3.1.共享数据问题
- 多线程同时读写同一个变量 / 数据结构
- 会产生竞争条件
- 结果=未定义行为(程序崩溃,数据错乱)

## 3.2.互斥锁(本章核心)
- std::mutex:独占锁,同一时间只有一个线程能锁
- lock():加锁,拿不到就阻塞
- unlock():解锁
- 绝对不要手动lock/ unlock,必须用RALL

## 3.3 RAII锁守卫
- std::lock_guard<std::mutex>:
   - 构造自动lock
   - 析构自动unlock
   - 最简单,最安全
- std::unique_lock<std::mutex>
   - 更灵活:可以延迟上锁,提前解锁,转移所有权
   - 更适合配合条件变量使用

## 3.4 死锁(deadlock)
- 两个线程互相持有对方需要的锁
- 解决方案:
    1.固定加锁顺序
    2.std::lock 一次性锁多个
    3.避免嵌套锁
    
## 3.5 保护共享数据的原则
- 最小锁范围
- 锁内不做耗时操作
- 不返回被保护数据的指针 / 引用

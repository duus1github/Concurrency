# 第5章 c++内存模型与原子操作
## 简介:
    是c++并发最底层,最硬核,决定无锁编程是否正确的一章,所有高级同步(mutex,condition_variable)底层都依赖本章内容.

### 1.内存模型
#### 1.1. 定义:
是c++对cpu如何读写内存,多线程之间的可见性的规则标准
#### 1.2. 原理
- 每个线程读写内存不是直接操作主存
- cpu有缓存,重排指令
- 多线程下可能导致读到旧值,指令乱序
#### 1.3 作用
定义线程之间:
- 什么操作是原子的
- 什么操作是对别的线程可见
- 什么顺序保证有效

### 2.数据竞争(最核心问题)
#### 2.1 定义
两个线程同时访问同一内存位置,且至少一个是写操作,且没有同步
#### 2.2 原理
读写操作不是原子的,导致读到写了一半的数据
#### 2.3 作用
无保护共享变量 = 未定义行为(程序漰溃,乱值,死循环)

### 3.原子操作
#### 3.1 定义
不可分割的操作,要么完全完成,要么完全没做
#### 3.2 原理
CPU提供单指令原子读写(如XCHG/CAS)
#### 3.3 作用
- 不是用mutex也能线程安全
- 实现无锁数据结构
- 实现高效同步

### 4.内存序(memory order)
#### 4.1 定义
控制cpu指令重排 + 多线程可见性规则.
#### 4.2 原理
CPU会乱序执行指令,内存序用来限制乱序
#### 4.3 作用
- 保证线程间操作顺序
- 保证写操作对其他线程可见
#### 4.4 最常用三种
**memory_order_seq_cst**
- 全局顺序一致
- 最简单,最安全,最慢
- 默认

**memory_order_relaxed**
- 只保证原子性
- 不保证顺序,不保证可见性
- 适合计数器

**memory_order_release/acquire**
- release:写之前的操作全部提交
- acquire:读之后能看到全部写
- 无锁编程标配

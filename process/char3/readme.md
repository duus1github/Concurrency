# 第三章 进程控制
## 1.进程的建立与运行
### 1.1 定义
    是正在运行的程序的实例,是linux系统资源分配(内存,cpu,文件)的最小单位
### 1.2.原理
内核为进程分配:
- pid:进程id,唯一标志符
- 代码段,数据段,堆,栈
- 文件描述表
- 进程优先级,状态
### 1.3.作用
1个客户端链接->分配1个子进程处理,实现并发服务

## 2.进程的建立--fork系统调用
### 2.1.定义
fork()是linux创建进程的唯一标准系统调用,调用一次,返回两次,分裂出子进程
### 2.2.原理
- 1.内核复制父进程的地址空间(写时复制copy-on-write,高效)
- 2.复制文件描述符,环境变量
- 3.父子进程从fork()下一行并发执行
### 2.3.作用
- 网络服务:父进程监听端口,子进程处理客户端
- 多任务并发执行
- 实现后台守护进程

## 3.进程的运行--exec系列函数
### 3.1.定义
exec系列函数,用新程序替换当前进程的代码段和数据段,让进程执行全新的程序.
### 3.2.原理
### 3.3.作用
- 子进程创建后,用exec执行具体业务,(如网络处理,命令,业务程序)
- fork + exec 是linux创建新任务的标准模式

## 4.数据和文件描述符的继承
### 4.1.定义
调用fork()创建子进程后:
1.全局变量,局部变量,堆内存都会复制一份给子进程
2.父进程所有已打开的文件描述符fd,子进程都会完整继承
3.父子进程共享同一个文件偏移量(文件读写位置同步)
### 4.2.原理
1.内存采用 写时复制COW
- 父子进程刚fork完,内存是共享的
- 任意方修改变量,内核才拷贝一份
2.文件描述表 完全复制
- 子进程拿到和父进程一样的fd编号
- 内核文件表是共享的,文件指针偏移量同一个
### 4.3.作用(网络编程重中之重)
1.网络服务经典模型:
父进程socket/bind/listen打开监听fd-fork子进程->子进程直接用继承的fd和客户端通信
2.串口/文件程序: 父打开设备,子进程直接读写
3.父子进程可以通过同一文件简单通信

## 5.进程终止 
### 5.1.定义
- exit(int status):库函数,刷新缓冲区,调用析构函数,清理资源,在终止进程
- _exit(int status):系统调用,直接粗暴终止,不刷新缓冲区,不做清理
### 5.2.原理
- exit:用户态完整收尾->进入内核终止
- _exit:直接诶进入内核销毁进程
### 5.3.作用
平时业务代码一律用exit
底层极简程序,嵌入式极简场景用_exit

## 6.进程同步
### 6.1.定义
父进程调用wait()阻塞等待子进程退出,并回收子进程资源(避免僵尸进程)
pid_t wait(int *status)
### 6.2.原理
1.父进程阻塞,直到任意一个子进程退出.
2.自动回收子进程PCB,防止僵尸进程
3.status可以获取子进程退出码
### 6.3.作用
网络服务,后台程序必备
父等待子进程干完活,在继续执行

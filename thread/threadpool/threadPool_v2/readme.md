# 线程池
## 1.定义
预先创建一批工作线程常驻内存,外部提交任务放入队列,空闲线程自动领取并执行任务;线程结束统一回收线程资源
## 2.解决什么问题
- 频繁创建销毁线程开销大:线程预先建好,复用线程,节省系统调用耗时
- 线程数量不可控,避免无限创建线程耗尽CPU,内存资源
- 任务统一调度:排队执行,均衡负载, 程序稳定性更高

## 3.核心组成
- 工作线程组:固定数量线程,循环等任务
- 任务队列:存放待执行的任务,先进先出
- 互斥锁:保护队列,防止多线程同时读写错乱
- 条件变量:无任务时线程休眠,有任务时唤醒线程,减少空轮询耗cpu
- 停止标记:安全终止线程池

## 4.运行流程
- 1.初始化:创建指定个数线程,线程阻塞等待任务
- 2.提交任务:任务推入队列,唤醒空闲线程
- 3.任务执行:线程取出任务执行,完成后继续等待新任务
- 4.销毁池:标记停止,唤醒所有线程,等待全部线程退出释放资源

## 5.使用场景
批量计算,异步IO,并发业务处理,定时任务调度
